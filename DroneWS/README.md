## 🚀 Overview

**DroneOS v0.0.3** is a fully configured drone-control and robotics base image for the **Raspberry Pi 5**, preloaded with:

- ROS 2 Jazzy  
- Navigation2 (Nav2)  
- SLAM Toolbox  
- MAVLink/MAVROS for **Cube Orange / ArduPilot**  
- Lidar and Depth
- Hotspot switching scripts  
- SSH enabled by default  
- Startup networking service  
- PiShrink auto-expand  

The `.img` file is shrink-optimized and will **auto-expand on first boot** to use the full SD card.

### 🔄 Toggle Network Mode

Switch **to Access Point mode**:

```bash
sudo toggleNetMode ap
```

Switch **to WiFi client mode**:

```bash
sudo toggleNetMode wifi
```

Connecting to your own wifi in client mode:

```bash
sudo nmcli dev wifi connect "SSID_NAME" password "PASSWORD"
```

---

### 🔧 Update Hotspot Configuration

Change SSID and settings:

```bash
sudo setupHotspot <prefix> <team#> <band> <password>
```

Where:

| Argument | Example | Description |
|----------|----------|-------------|
| `<prefix>` | `drone` | First part of SSID |
| `<team#>` | `1234` | Four-digit ID |
| `<band>` | `2.4` or `5` | WiFi band |
| `<password>` | `password` | WPA2 password (≥ 8 chars) |

After running, you will be prompted to choose a WiFi channel.

## 💾 Flashing Instructions

### ⚠️ Important Note

Raspberry Pi Imager **does NOT support `.7z` files**.  
You must **extract the 7z file first** to access the `.img`.

---

### **Windows**

1. Install **7-Zip**  
2. Extract the image:
   ```text
   DroneOS_v0.0.3.7z  →  DroneOS_v0.0.3.img
   ```
3. Open **Raspberry Pi Imager**  
4. Click *Use custom* and select the `.img`  
5. Flash to SD card  
6. Insert SD → boot Pi  
7. Wait 1–2 minutes for autoexpand  
8. Reboot if prompted

---

### **Linux**

```bash
7z x DroneOS_v0.0.3.7z
sudo dd if=DroneOS_v0.0.3.img of=/dev/sdX bs=4M status=progress
sync
```

## ✈️ MAVROS Quick Start

Connect Cube Orange via USB.

### Example: 57600 baud over USB ACM0

```bash
ros2 launch mavros apm.launch fcu_url:="serial:///dev/ttyACM0:57600"
```

Connect Cube Orange via USB and stream to ground station. You need the IP address of the computer your ground station is on. On my computer this was 172.22.11.40 as I was connected over Ethernet.

```bash
ros2 launch mavros apm.launch fcu_url:="serial:///dev/ttyACM0:57600" gcs_url:="udp://@172.22.11.40:14550"
```

### Verify communication:

```bash
ros2 topic list
ros2 topic echo /mavros/state
```

### Read Battery

```bash
ros2 topic echo /mavros/battery
```

### Arm and Disarm

```bash
ros2 service call /mavros/cmd/arming mavros_msgs/srv/CommandBool "{value: true}"
```

```bash
ros2 service call /mavros/cmd/arming mavros_msgs/src/CommandBool "{value: false}"
```

## ✈️ MAVProxy Quick Start

Use MAVProxy on a Raspberry Pi to bridge a Cube Orange (USB) to Mission Planner over Ethernet or Wi-Fi.

### Network Topology

    Cube Orange ──USB──▶ Raspberry Pi ──Ethernet / Wi-Fi──▶ Computer (Mission Planner)

### Launch MAVProxy on Raspberry Pi

NOTE:
You must use the IP address assigned to your computer by the Raspberry Pi / network
(not the Raspberry Pi’s own IP address).

Example:
If your computer’s IP address is 10.12.34.72, run the following on the Raspberry Pi:

```bash
mavproxy.py \
   --master=/dev/ttyACM0 \
   --baudrate 115200 \
   --out udp:10.12.34.72:14550
```

- /dev/ttyACM0  → Cube Orange USB device
- 10.12.34.72   → Computer running Mission Planner
- 14550         → MAVLink UDP port

To find your computer’s IP address:
- Windows: ipconfig
- Linux / macOS: ip a  or  ifconfig

### Launch Mission Planner (Computer)

1. Open Mission Planner
2. Set connection type to UDP
3. Set port to 14550
4. Click Connect

Mission Planner should connect immediately.
